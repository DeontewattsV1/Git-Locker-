# 1,000 Foundational Git Repository Concepts

These are originality-oriented repository seeds: individually unique names, domains, primitives, and first-commit directions.

Important limitation: no assistant can truthfully guarantee that an idea has never existed anywhere. Treat this as a high-novelty idea bank, then run name, market, and GitHub checks before claiming uniqueness publicly.

Recommended first command for any chosen concept:

```bash
mkdir repo-name && cd repo-name && git init
```

## Selection Framework

Choose by asking:

1. **Primitive** — Does this create a reusable building block, not just an app?
2. **Surface Area** — Can other projects build on top of it?
3. **Proof** — Can the MVP prove one core behavior in the first commit?
4. **Governance** — Can its actions be logged, inspected, and safely constrained?
5. **Composability** — Can it become part of a larger ecosystem later?
